你是 Replit 里的资深全栈开发者 + 产品设计师，帮我在本项目里从 0 搭出一个可运行的 Web demo：

## 产品定位

产品名：**Prompt Imagination Studio**

用途：不是给最终答案，而是给「写 Prompt / 想点子」之前做调味的。  
用户来这里，是为了先拿到一些：

- Magic Words（有方向感的关键词 + 示例片段）
- Tension Seeds（有画面感的张力句子 + 延展问题）

之后再把这些内容丢回 ChatGPT / Claude 去继续创作。

核心原则：

- ❌ 不做复杂控制面板，不堆满 slider
- ✅ 操作步骤尽量 1–2 步就有结果
- ✅ UI 简洁但有一点「性感」：参考 Google TextFX，见3张附件图片（顶部模式卡片 + 大标题 + 右侧输入 + 下方结果卡片）

---

## 整体结构

请在 Replit 里选择一个你最熟悉、搭建最快的 Web 技术栈（前后端你自行决定，只要在 Replit 一键 Run 就能打开一个网页）。

页面可以是：

- 单页应用，有「Mode 切换」的交互；或者
- 很浅的多路由（例如 `/magic`、`/tension`）

你根据实现便利性自行选择，但要保证两种 Mode 在同一个站点里，不要拆成两个项目。

顶部：模式切换卡片区  
左侧：当前 Mode 的标题 + 描述  
右侧：输入区（文本输入 + 少量选项）  
下方：输出区（结果卡片列表 + 复制按钮）

视觉风格参考 TextFX（见3张附件图片）：大块背景色、卡片、圆角按钮、对齐清晰即可，不需要花哨动画。

## Mode 1：Magic Word Atelier

目标：给一个具体任务配几小撮「魔法词」。

左侧标题 & 描述（可微调文案）：

- 标题：`Magic Word Atelier`
- 副标题：`给任务配一小撮魔法词`

右侧输入区包含：

- 一个多行文本输入，用来描述任务和背景：
  - placeholder 示例：
    - 「任务：给产品负责人看的《Claude Code Sub-Agent》产品分析。」
    - 「风格倾向：偏结构 + 禅意，不要 PR 文案。」
- 一个可选的「风格/意图」单行输入（如果你觉得和上面合并更简洁，也可以合并）
- 一个温度或「发散程度」控制（可以是 slider 或几个预设按钮，高/中/低），文案用你觉得合适的方式表达
- 一个明显的 `RUN` 按钮

输出区：

- 以卡片列表的形式展示 3–5 条 Magic Word，每条包括：
  - Magic Word（短词/短语）
  - 一句解释：这个词会把 AI 的输出往哪个方向拉
  - 一段「示例 Prompt 片段」：展示这个词如何嵌进一个更长的 Prompt 中
- 每张卡片上有一个「复制片段」按钮，可以一键复制示例 Prompt 片段到剪贴板

功能上，先用**假数据/模拟函数**完成端到端体验：  
点击 RUN → 输出区刷新出几张合理的示例卡片。  
请把生成逻辑封装在一个函数里（例如 `generateMagicWords(...)`），并在代码里用注释标记清楚：之后我会在这里接 LLM。

## Mode 2：Tension Seeds Studio

目标：围绕「主题 + 张力轴」，生成有画面感的张力句子 + 延展问题，帮助构思 demo / 文章切口 / 产品叙事。

左侧标题 & 描述（可微调文案）：

- 标题：`Tension Seeds Studio`
- 副标题：`给创意和 demo 打一针张力种子`

右侧输入区包含：

- 主题输入框（单行）：例如 placeholder `主题：Prompt 即协议`
- 张力轴输入（多行，每行一条）：例如
  - 「禅意 vs 工程」
  - 「严肃专业 vs 玩具感」
- 发散程度控制（可以沿用上一个 Mode 的交互）
- 一个 `RUN` 按钮

输出区：

- 展示 3–5 条「张力种子」，每条包括：
  - 一句像推文标题那样有画面的 Seed Sentence
  - 1–2 个延展问题（Follow-up questions），用于进一步发散方向
- 同样先用**假数据/模拟函数**完成体验，生成逻辑封装在一个函数（例如 `generateTensionSeeds(...)`），用注释标记未来替换为真实 LLM 调用的位置。

## 交互 & 结构要求（抽象级别）

- 顶部模式切换：
  - 两张卡片或 Tab：「Magic Word」「Tension Seeds」
  - 当前模式高亮，切换时更新：
    - 页面左侧标题/描述
    - 右侧输入区域
    - 下方输出区域
- RUN 时：
  - 读取当前模式的输入，调用对应的生成函数
  - 清空旧结果并展示新卡片
- 复制行为：
  - Magic Word 卡片提供一键复制功能；如果实现上方便，也可以给 Tension Seeds 的 Seed Sentence 加复制按钮

请你自行选择合适的组件结构和代码组织方式，但要保证：

1. 代码结构清晰，文件数量可控，便于我阅读和修改；
2. 有适量注释，说明关键模块的职责，尤其是未来要接 LLM 的部分；
3. 初版跑起来后，即使完全是模拟数据，也能给人「已经是一个可用的上游调味工作台」的感觉。
